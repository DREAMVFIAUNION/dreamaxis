# DreamAxis v0.2.0-preview — Desktop Operator Update

DreamAxis is no longer just a local-first foundation release.

With `v0.2.0-preview`, DreamAxis now presents a much clearer product shape:

**a local-first desktop operator with a specialized repo-copilot lane**

## What is new

- real **Windows host Desktop Runtime**
- approval-gated desktop actions
- grounded desktop targets in chat
- runtime-backed audit trails for desktop execution
- refreshed operator-console screenshots for GitHub

## Confirmed desktop actions

The current preview has already validated:

- focus window
- launch app
- press hotkey
- type text
- click

This is still a controlled system, not a black-box autonomous agent. State-changing actions stay gated and visible.

## Why this update matters

Most AI tooling still collapses everything into a generic chat surface. DreamAxis keeps pushing in a different direction:

- local-first instead of cloud-first
- execution-first instead of prose-first
- evidence and audit instead of opaque autonomy
- self-hosted keys and data instead of central lock-in

## What remains true

DreamAxis still keeps its core boundaries:

- no-signup by default
- provider keys stay self-hosted
- runtime-backed evidence stays central
- repo/code changes remain proposal-only

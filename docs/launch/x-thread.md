# DreamAxis launch thread

## Post 1

I just open-sourced **DreamAxis**.

DreamAxis is a **local-first agent execution platform** for self-hosted AI workflows.

Not another hosted chatbot.
Not a forced-signup SaaS.
More like a reusable execution foundation built around runtimes, skills, and knowledge.

GitHub: https://github.com/DREAMVFIAUNION/dreamaxis

## Post 2

The default path is intentionally simple:

- `AUTH_MODE=local_open`
- no public signup
- your provider keys stay self-hosted
- your knowledge files stay on your disk
- your metadata stays in your PostgreSQL

## Post 3

DreamAxis already includes:

- **CLI Runtime**
- **Browser Runtime (Playwright)**
- **OpenAI-compatible provider connections**
- **builtin skill packs**
- **builtin knowledge packs**
- **runtime/session/execution visibility**

## Post 4

What I wanted was a system where:

- local execution matters
- model gateways stay portable
- skills become reusable assets
- knowledge compounds over time
- the product is more than a pretty chat screen

That is the direction DreamAxis is taking.

## Post 5

This first public release is not pretending to be a complete multi-agent OS.

It is a solid v0.1.0 foundation for:

- local-first workflows
- runtime-backed execution
- self-hosted AI operations
- future role-aware orchestration

## Post 6

If that sounds useful, try it, break it, and open issues.

The first milestone is `v0.1.x polish`, so feedback on:

- docs
- provider UX
- runtime ergonomics
- skills / knowledge polish

...is especially valuable right now.

# Support

## Where to ask for help

Use GitHub issues for:

- bug reports
- feature requests
- documentation gaps
- local setup problems

Please use the issue templates when possible.

## Before opening an issue

Check:

- [README.md](./README.md)
- [docs/development.md](./docs/development.md)
- [docs/deployment-modes.md](./docs/deployment-modes.md)
- [docs/browser-runtime.md](./docs/browser-runtime.md)

## What to include

Helpful support details:

- operating system
- Docker or non-Docker setup
- `AUTH_MODE`
- whether Browser Runtime is enabled
- exact error text
- relevant logs
- reproduction steps

## Secrets and privacy

Please redact:

- API keys
- tokens
- database credentials
- personal data

If the issue looks security-sensitive, follow [SECURITY.md](./SECURITY.md) instead of posting full details publicly.
